# 제품 요구사항 정의서 (PRD) - 링크트리 클론 서비스 ("My Link")

본 문서는 사용자가 자신만의 멀티 링크 페이지를 구축하고 관리할 수 있는 서비스 **"My Link"**의 핵심 제품 요구사항 정의서(PRD)입니다. 본 문서는 핵심 기능인 사용자인증, 프로필관리, 링크블록 관리, 퍼블릭페이지에 초점을 맞추어 작성되었습니다.

---

## 1. 프로젝트 개요

### 1.1 서비스 목적
- **"My Link"**는 사용자가 여러 플랫폼에 분산된 자신의 온라인 채널과 웹사이트 링크를 단 하나의 프로필 랜딩 페이지로 통합하여 방문자에게 제공할 수 있는 멀티 링크 빌더 서비스입니다.

### 1.2 대상 사용자
- 소셜 미디어 프로필에 단 하나의 링크만 게시할 수 있는 환경(인스타그램, 틱톡 등)에서 다수의 링크를 홍보하고자 하는 개인 크리에이터, 브랜드, 비즈니스 및 일반 사용자.

### 1.3 핵심 가치
1. **극도의 단순성**: 복잡한 설정 없이 가입부터 링크 배포까지 단 몇 단계만으로 완료할 수 있는 최적의 사용성.
2. **직관적인 비주얼**: 관리 화면에서 직접 설정한 결과가 모바일 화면에 어떻게 반영되는지 즉시 확인 가능한 환경 제공.

---

## 2. 핵심 기능 범위

"My Link"의 핵심 기능은 다음 4가지 요소로 구성됩니다.

1. **사용자 인증 (User Authentication)**: 회원가입, 로그인 및 개인 고유 URL 식별자 생성.
2. **프로필 관리 (Profile Management)**: 랜딩 페이지 상단에 노출될 프로필 이미지, 닉네임, 간단한 소개 작성.
3. **링크블록 관리 (Link Block Management)**: 방문자를 특정 URL로 이동시키는 링크 블록의 생성, 편집, 정렬 및 활성화 여부 조절.
4. **퍼블릭 페이지 (Public Page)**: 외부 방문자에게 공개되는 모바일 최적화 랜딩 페이지.

---

## 3. 기능별 상세 설명

### 3.1 사용자인증 (User Authentication)
- **설명**: 서비스 이용 및 나만의 고유 링크 관리를 위한 계정 체계와 접속 주소를 정의합니다.
- **상세 요구사항**:
  - **가입 및 로그인**: 이메일 및 비밀번호를 통한 기본 회원가입 및 로그인 기능을 제공합니다.
  - **고유 URL 선점**: 가입 단계 또는 가입 후 설정에서 고유 식별자(Username)를 지정하여 자신만의 접속 주소(예: `mylink.at/{username}`)를 가집니다.
  - **중복 검사**: 데이터베이스 내 이미 존재하는 Username은 사용할 수 없도록 실시간 검증을 수행합니다.

### 3.2 프로필관리 (Profile Management)
- **설명**: 퍼블릭 랜딩 페이지 상단에 표시되는 사용자의 아이덴티티 정보를 관리합니다.
- **상세 요구사항**:
  - **프로필 이미지(아바타)**: 사용자가 아바타로 사용할 이미지를 업로드할 수 있는 영역을 제공합니다. (크기 조정 및 원형 크롭 기능 포함)
  - **프로필명(닉네임)**: 퍼블릭 페이지 최상단에 강조되어 노출될 닉네임을 설정합니다. (글자 수 제한: 최대 30자)
  - **소개글(Bio)**: 프로필명 아래에 짧게 들어갈 서술형 소개글을 작성할 수 있습니다. (글자 수 제한: 최대 80자)

### 3.3 링크블록 관리 (Link Block Management)
- **설명**: 랜딩 페이지의 핵심 요소인 바로가기 링크 버튼들을 생성하고 관리하는 빌더 환경입니다.
- **상세 요구사항**:
  - **링크 블록 추가**: "새 블록 추가" 버튼 클릭 시 새로운 제목(Title)과 연결 URL을 입력하는 카드가 생성됩니다.
  - **텍스트 및 URL 편집**: 각 링크 블록마다 방문자에게 보여줄 노출 명칭과 실제 연결될 아웃링크 URL을 기재합니다.
  - **순서 변경**: 마우스 드래그 앤 드롭 또는 상하 이동 버튼을 사용하여 노출 순서를 편리하게 변경합니다.
  - **노출 활성화 토글**: 링크를 완전히 삭제하지 않고도 퍼블릭 페이지에서 숨기거나 보여줄 수 있는 ON/OFF 토글 스위치를 지원합니다.
  - **실시간 프리뷰**: 관리 화면 한편에 가상의 모바일 화면(Preview)을 배치하여 링크블록의 수정 사항이 실시간으로 동기화되어 보이도록 설계합니다.

### 3.4 퍼블릭페이지 (Public Page)
- **설명**: 사용자의 개별 URL(`mylink.at/{username}`)로 누구나 접속하여 볼 수 있는 외부에 공개된 랜딩 페이지입니다.
- **상세 요구사항**:
  - **모바일 퍼스트 레이아웃**: 모든 화면 크기(특히 스마트폰 모바일 뷰)에서 레이아웃이 깨지지 않도록 반응형으로 설계합니다.
  - **정보 노출 구조**: 최상단에 프로필 이미지, 닉네임, 소개글이 순서대로 위치하며, 그 아래로 활성화 상태인 링크 블록들이 직관적인 버튼 형태로 나열됩니다.
  - **클릭 인터랙션**: 링크 블록 버튼을 터치/클릭하면 사용자가 입력한 해당 외부 URL로 새 창 또는 현재 창에서 이동합니다.

| :--- | :--- | :---: | :---: |
| **사용자 인증** | 이메일 기반 회원가입 및 로그인 | **High (P0)** | Phase 1 |
| **사용자 인증** | 고유 Username 설정 및 URL 중복 검사 | **High (P0)** | Phase 1 |
| **프로필 관리** | 아바타 이미지 업로드 및 삭제 | **Medium (P1)** | Phase 2 |
| **프로필 관리** | 프로필명(닉네임) 및 소개글 입력/저장 | **High (P0)** | Phase 1 |
| **링크블록 관리** | 링크 블록 생성, 타이틀 및 URL 지정 | **High (P0)** | Phase 1 |
| **링크블록 관리** | 링크 블록 ON/OFF 활성화 토글 스위치 | **High (P0)** | Phase 1 |
| **링크블록 관리** | 링크 블록 드래그 앤 드롭 정렬 | **Medium (P1)** | Phase 2 |
| **링크블록 관리** | 관리 대시보드 내 실시간 모바일 프리뷰 | **High (P0)** | Phase 1 |
| **퍼블릭 페이지** | 모바일 최적화 반응형 랜딩 페이지 | **High (P0)** | Phase 1 |
| **퍼블릭 페이지** | 활성화된 링크 블록 렌더링 및 클릭 이동 | **High (P0)** | Phase 1 |
